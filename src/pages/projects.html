Web App integration with CRM

This is a full featured web app I built with Vue in an existing CRM system for a client while working at Breankman and Company. Everything on this page (except for the map and title bars) is dynamically generated based on data fetched from the server.

This app solves a Multiple Traveling Salesman Problem in which several techs needed to be dispached to accomplish multiple jobs while optimizing for time. However, this specific implementation was more complicated than the classic case because there were custom limitations and requirements that had to be met. To name a few: some jobs had to be done on certain days and times, each job had its own set of tasks and each tech had a list of skills that specified which jobs they could do, some jobs had higher priorety ranking, the time spent on each job needed to be predicted based on tasks and time metrics associated with the job.

Routes are automatically generated by a complex algorithm which I designed. In adition, controls were implemented to allow the user to drag and drop jobs to different days and to reorder them. Data changes are tracked and relayed to the server to sync wtih the CRM.

Google Maps APIs were used for the map and route planning functionality

I did not make all of the design choices for this project, but the front end and server side functionality was primarily implemented by myself.
